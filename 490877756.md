# Problem Description
## Summary

Storing a shared wasm struct into a JS `SharedArray` crashes with a DCHECK failure in debug builds and throws an incorrect `TypeError` in release builds. The CSA `SharedValueBarrier` fast path is missing checks for `WASM_STRUCT_TYPE` and `WASM_ARRAY_TYPE`, causing already-shared wasm objects to fall into a slow path that cannot handle them.
